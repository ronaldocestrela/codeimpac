## Full Instructions for AI Agent Implementation

# Project Overview
This project is a SaaS platform that:
- Connects to GitHub
- Analyzes developer contributions (commits and pull requests)
- Identifies approved pull requests using GitHub review states
- Uses LLM to generate professional executive reports for leadership

# Product Premise (Non-Negotiable)
- Developers authenticate with GitHub
- Developers select one or more repositories to analyze
- The system ingests commits and pull requests for the selected repositories
- The system classifies PR approval status based on GitHub review data
- The system generates a leadership-ready report with evidence and metrics

# Backend Rules

- Follow Clean Architecture strictly
- No business logic in controllers
- Use MediatR pattern (commands/queries)
- LLM must be abstracted via interface

---

# AI Integration Rules

- Never call LLM directly in controller
- Use IAIOrchestrator
- Always log prompt and response
- Use retry policies
- Build prompts with structured contribution inputs (commits + approved PRs)
- Require deterministic preprocessing before LLM (filtering, grouping, metrics)
- Always keep traceable evidence IDs in generated insights

# Contribution Analysis Rules

- Commits and pull requests are first-class entities in the domain model
- Approved PR classification must use GitHub review state (`APPROVED`) and be persisted
- PR analysis must include status dimensions: open, closed, merged, approved
- Keep author, repository, timestamps, reviewers, and links for auditability

# Executive Report Requirements

- Reports must target managers/supervisors (clear, concise, outcome-oriented)
- Include period, repositories analyzed, and developer scope
- Include objective metrics (volume, throughput, review outcomes, cycle indicators)
- Include qualitative highlights generated by LLM with supporting evidence
- Include risks/blocked items and recommended next actions
